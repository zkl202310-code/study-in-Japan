# 日本大学院出愿 DDL + 门槛追踪器（jp-grad-ddl-tracker）

> 一句话：把"各日本大学院（先做 Top 30 国公立 · **教育系** · 修士）当年最新**出愿截止日 + 语言/推荐信/事前连络硬门槛**"做成可**动态筛选 + 倒排时间线 + 新要项发布即时推送**的工具。核心价值是一份**别人读不动的日语募集要項结构化库**，不是 AI 对话。

- **创建日期**：2026-06-05
- **当前阶段**：📐 规划完成，准备进入 **第 0 阶段（周末验证切片）**
- **创始人壁垒**：N1 + 教育学硕士 + 日语本地化经验 → 能读懂、肯录入 1700+ 份日语募集要項 PDF（绝大多数独立开发者读不动 / 不愿读）。这道门槛就是护城河。

## 快速开始（第 0 阶段，周末能做完）

目标：**用最便宜的方式验证"有没有真需求"**，不是上来就写完整系统。

1. 打开 `data/universities.seed.json`，按 [docs/01](docs/01-数据规则与Schema.md) 的字段，手工录入 **8 所顶尖国公立的教育系研究科修士**募集要項（東大/京大/阪大/东北/名大/一桥/筑波/广岛）。**全部数据必须来自官方 PDF**，见 [docs/02](docs/02-质量保证.md) 红线。
2. 做一个**静态 Next.js 单页**展示这 8 条 + 纯前端筛选器，部署到 Vercel（绑自定义域名）。
3. 发一条小红书《日本教育学大学院 2027 出愿 DDL 速查（持续更新）》带链接。
4. **看 48 小时数据**：收藏 < 20 或没人评论"我的学校呢" → 暂停重估（最便宜的止损点）。

## 三条核心纪律（详见各文档）

1. **范围锁定**：只做 大学院 · 修士 · 教育系 · 留学生入試 · 主募集期 · Top30 校。能砍的维度全砍。
2. **数据即命**：信任是唯一资产。**绝不编造/猜测任何 DDL 或要件**，必须有官方 `source_url` + `last_verified` 日期。
3. **产品内不调 LLM**：所有筛选/时间线都是确定性计算 → 不触发生成式 AI 备案；收款走爱发电 / ToB，绕开个人主体收费墙。
